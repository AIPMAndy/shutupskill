# shutupskill

让 AI 不废话 — 通过提示词优化改善输出风格。

## 问题

GPT-5.4/Kimi 在 OpenClaw 中的通病：
- ❌ **啰嗦** — 一句能说完的事，输出 200+ Token
- ❌ **工具调用弱** — 参数缺失、串行低效
- ❌ **半途而废** — 遇错就停，推给用户
- ❌ **Token 浪费** — 冠词、客套话、犹豫表达，全是无效消耗

## 解决

通过提示词优化，改变 AI 输出风格和工作流程：

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| 单次输出 Token | 200-500 | **50-150**（降 60-70%） |
| 输出风格 | 啰嗦、客套 | **简洁、直接** |
| 工作流程 | 问完再做 | **先做后说** |
| 输出句数 | 5-10 句 | **≤ 3 句** |

## 为什么能省 Token？

废话 = 浪费的 Token = 浪费的钱。

```
优化前（178 Token）：
"好的，让我先帮您读取一下这个配置文件，然后我会仔细分析其中
可能存在的性能瓶颈问题。我注意到这个文件有 100 行代码，其中
包含了一些可能影响性能的部分。您希望我优先优化哪个方面呢？"

优化后（31 Token）：
[直接执行]
"配置有 3 处瓶颈。已优化。响应时间降 60%。"
```

**通过减少冗余表达，可显著降低输出 Token。**

## 核心优化

不废话 ≠ 功能缩水。通过提示词引导：

- **先做后说** — 直接执行，不问"您希望怎样"
- **工具并行** — 能同时调用的工具不串行，提升效率
- **错误重试** — 引导模型遇错尝试多种方案
- **输出规范** — 明确的输出格式和禁用词规则

**原理：通过结构化提示词，引导模型改变输出风格和工作流程。**

## 功能

通过在 agent 的 SOUL.md 中注入优化规则，引导 AI 改变输出风格：
- 先做后说，减少询问
- 工具并行调用提示
- 错误重试引导
- 极简输出规范


## 使用

```bash
/shutup --all                    # 优化所有 agent
/shutup --agent dev              # 优化指定 agent
/shutup --upgrade                # 升级已优化的 agent 到最新版本
/shutup --status                 # 查看所有 agent 的优化状态
/shutup --diff                   # 预览会注入的内容
/shutup --template-only          # 查看完整优化模板
/shutup --restore                # 恢复所有备份
/shutup --restore --agent dev    # 恢复指定 agent
```

## 工作原理

这是一个**提示词注入工具**，通过在 `~/.openclaw/agents/<agent>/workspace/SOUL.md` 末尾追加优化规则来改变 AI 行为。

**能做到的**：
- ✅ 改变输出风格（少废话）
- ✅ 引导工作流程（先做后说）
- ✅ 提供检查清单
- ✅ 版本管理和升级
- ✅ 自动备份（保留最近 5 个版本）

**做不到的**：
- ❌ 保证任务完成率
- ❌ 强制工具精准度
- ❌ 让弱模型变成强模型

**效果取决于模型本身的能力。**

## 优化维度

| 维度 | 规则 | 效果 |
|------|------|------|
| 工作流程 | 先做后说，不问再做 | 减少来回对话轮次 |
| 工具调用 | 检查清单 + 优先级 + 并行提示 | 引导更好的工具使用 |
| 输出效率 | ≤ 3 句话 + 删冠词/客套话 | **输出 Token 降 60-70%** |
| 错误处理 | 引导尝试多种方案 | 减少直接放弃的情况 |

### 输出规则

**删除（省 Token）：**
- 冠词（"一个"、"这个"）
- 语气填充词（"那么"、"其实"）
- 客套话（"希望能帮到您"）
- 犹豫性表达（"可能"、"也许"）

**保留（保质量）：**
- 技术术语、代码块、报错信息
- 短句、碎片句

**优先短同义词：**
- 「大」不是「庞大」 — 省 1 Token
- 「修」不是「实施解决方案」 — 省 3 Token
- 「查」不是「进行检查」 — 省 2 Token

**推荐句式：**
```
[问题][动作][原因]。[下一步]。

"配置错误。已修复。原因：缺 API key。重启生效。"
"性能瓶颈在第 45 行。已优化。响应时间降 60%。"
```

## Token 节省效果

根据使用场景和模型特性，优化后可能节省 **50-70% 输出 Token**。

**注意**：
- 实际效果取决于模型本身的啰嗦程度
- 本来就简洁的模型（如 Claude）提升有限
- 啰嗦的模型（如 GPT-5.4/Kimi）效果更明显

## 适用场景

### ✅ 适合

- **啰嗦的模型**：GPT-5.4、Kimi、Qwen 等
- **高频使用**：每天大量对话，Token 成本敏感
- **任务型对话**：编程、数据分析、文件操作等
- **追求效率**：希望 AI 少废话、直接干活

### ⚠️ 不适合

- **本来就简洁的模型**：Claude Opus 4.6（已经很简洁）
- **需要详细解释**：教学、科普、需要展开说明的场景
- **创意写作**：需要丰富表达和文学性的内容
- **情感交流**：需要温暖、共情的对话

## FAQ

### Q: 会影响 AI 的能力吗？
A: 不会。这只是改变输出风格，不改变推理能力。就像让一个人说话简洁点，不会让他变笨。

### Q: 可以随时恢复吗？
A: 可以。每次优化前都会自动备份，运行 `/shutup --restore` 即可恢复。

### Q: 如何升级到新版本？
A: 运行 `/shutup --upgrade --all` 即可升级所有已优化的 agent。

### Q: 如何查看哪些 agent 已优化？
A: 运行 `/shutup --status` 查看所有 agent 的优化状态。

### Q: 优化后 AI 变得不友好了怎么办？
A: 这个工具适合任务型对话。如果你需要更温暖的交流，可以恢复备份或不使用此工具。

### Q: 真的能省 60-70% Token 吗？
A: 取决于模型。啰嗦的模型（如 GPT-5.4/Kimi）效果明显，简洁的模型（如 Claude）提升有限。
